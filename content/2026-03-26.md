## Resources

### 1. Put Claude on Autopilot: /loop and /schedule Skills

**Source:** [Rick Hightower on Medium](https://medium.com/@richardhightower/put-claude-on-autopilot-scheduled-tasks-with-loop-and-schedule-built-in-skills-43f3be5ac1ec)
**Time:** ~5 min read
**Category:** New Feature — Automation

**Why it's relevant:** `/loop` is brand new (March 2026) and tailor-made for consultant workflows — think polling a deployment, babysitting a CI pipeline, or auto-checking PR status every 10 minutes while you work on something else. `/schedule` goes further with persistent cloud-based recurring tasks.

**Key Takeaway:** Use `/loop 10m /review-pr` to have Claude automatically review open PRs every 10 minutes during your session. For tasks that need to survive restarts, use `/schedule` with a cron expression instead. Both free you to focus on high-value work while Claude handles monitoring.

### 2. Claude Code for Data Analysis

**Source:** [Vincent Codes Finance](https://vincent.codes.finance/posts/claude-code-data-analysis/)
**Time:** ~5 min read
**Category:** Core Workflow — Data Analysis

**Why it's relevant:** Data analysis is bread and butter for BCG work. This guide covers how to set up Claude Code as a data analysis partner — from pointing it at your datasets to defining specialized sub-agents for figure generation, statistical analysis, and data cleaning.

**Key Takeaway:** Define your data locations and schema in CLAUDE.md so Claude knows where everything is from session start. Break analysis into focused steps (load, validate, preprocess, analyze) rather than asking for the whole pipeline at once — you'll get dramatically better results and maintain the line-by-line review discipline that client work demands.

### 3. Claude Code Best Practices (Official Docs)

**Source:** [Anthropic — Best Practices](https://code.claude.com/docs/en/best-practices)
**Time:** ~5 min read
**Category:** Evergreen — Workflow Optimization

**Why it's relevant:** The official best practices doc is the authoritative reference and was recently updated. It covers the Research-Plan-Execute-Review-Ship pattern, context management with `/clear` and `/compact`, and how to structure CLAUDE.md files with nested directory-specific configs — all directly applicable to managing complex consulting codebases.

**Key Takeaway:** Use **Plan Mode** (Shift+Tab) before every non-trivial task. The 2 minutes spent planning saves 20 minutes of rework. Use `/clear` between unrelated tasks to keep context clean, and nest CLAUDE.md files in subdirectories (e.g., `/analysis`, `/scripts`) for focused context per workstream.

## Quick Win

**Try this today:** Run `/init` in your current project if you haven't already, then manually edit the generated CLAUDE.md to add:

```markdown
## Data Locations
- Raw data: ./data/raw/
- Processed: ./data/processed/
- Outputs: ./outputs/

## Conventions
- All scripts in Python 3.11+
- Use pandas for tabular data
- Charts use matplotlib with BCG brand colors
- Always validate row counts after joins
```

This gives Claude persistent memory about your project structure and standards, eliminating the 10-15 minutes you'd otherwise spend re-explaining context each session. It takes 2 minutes and pays for itself immediately.
