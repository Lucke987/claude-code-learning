## Resource 1: Claude Code for Data Analysis — End-to-End Workflow

**Source:** [Claude Code for Data Analysis – Vincent Codes Finance](https://vincent.codes.finance/posts/claude-code-data-analysis/)
**Time:** ~5 min read
**Category:** Directly relevant to your data work

### Why This Matters for You
This guide walks through using Claude Code specifically for data analysis projects — from setting up `CLAUDE.md` with dataset paths and schema descriptions, to having Claude write pandas transformations and generate plots. Instead of uploading CSVs to the browser, you can point Claude Code at local files and let it iterate in your terminal.

### Key Takeaway
**Put your data context in `CLAUDE.md`.** Add a section like:
```
## Data
- Raw data lives in `data/raw/`
- Use pandas for all transformations
- Save plots to `figures/` as PNG
- Output format: Excel with formatted headers
```
This eliminates repeated explanations and lets Claude produce correct code on the first try. For large datasets, have Claude write a processing script rather than loading everything into context.

## Resource 3: 10 Claude Code Productivity Workflows for 2026

**Source:** [Claude Code Tips: 10 Real Productivity Workflows](https://www.f22labs.com/blogs/10-claude-code-productivity-tips-for-every-developer/)
**Time:** ~5 min read
**Category:** Workflow optimization

### Why This Matters for You
This article covers structured workflows that map well to consulting work:

- **Custom slash commands** — Create reusable prompts in `.claude/commands/`. For example, a `/analyze-data` command that always follows your firm's analysis framework, or `/format-excel` for consistent output formatting.
- **Worktree-based parallel work** (`--worktree` / `-w` flag) — Work on multiple client deliverables simultaneously without branch conflicts. Claude runs in an isolated copy of your repo.
- **Headless mode** (`-p` flag) — Run Claude from scripts for batch automation. Perfect for nightly data processing or scheduled report generation.
- **`/rewind`** — Roll back conversation and code to earlier checkpoints. Safe experimentation when trying different analysis approaches.

### Key Takeaway
Create one custom command this week. Put a file at `.claude/commands/quick-analysis.md` with:
```
Analyze the dataset at $ARGUMENTS:
1. Load and describe the shape, dtypes, and missing values
2. Generate summary statistics
3. Create 3 key visualizations saved to figures/
4. Write a brief findings summary to output/findings.md
```
Then run `/project:quick-analysis data/client_survey.csv` and watch Claude do the entire pipeline.

## Quick Win — Apply This Today

**Set up your `CLAUDE.md` for your current project.** Run `/init` in your project directory, then add:

1. **Data locations** — Where raw and processed data lives
2. **Output conventions** — File formats, naming patterns, output directories
3. **Tech stack** — "Use pandas, not polars. Use openpyxl for Excel. Use matplotlib for charts."
4. **Quality rules** — "Always include data validation. Log row counts before and after transformations."

This single file will make every Claude Code session 2-3x more productive because Claude stops guessing and starts following your conventions from the first prompt. Takes 5 minutes to set up, saves hours over the week.
