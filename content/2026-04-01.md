### 2. Claude Code Hooks: A Practical Guide to Workflow Automation (DataCamp)

- **URL:** https://www.datacamp.com/tutorial/claude-code-hooks
- **Time estimate:** 5 min read
- **Category:** Workflow Automation

**Why it's relevant:** Hooks are the most underleveraged Claude Code feature for consultants. They let you enforce deterministic behavior — unlike CLAUDE.md instructions (which are advisory), hooks *always* execute. This matters when you need guaranteed formatting, linting, or safety checks.

**Practical examples for your workflow:**

- **Auto-format on save:** A `PostToolUse` hook that runs `black .` after every Python file edit — your code is always clean without thinking about it
- **Block dangerous operations:** A `PreToolUse` hook that prevents writes to `data/production/` — safety guardrail for client data
- **Auto-lint on stop:** A `Stop` hook that runs `pylint` when Claude finishes — catches issues before you even review

**How to set one up (30 seconds):**
Ask Claude: *"Write a hook that runs black after every Python file edit"* — it will create the `.claude/hooks/` config for you automatically.

**Key Takeaway:** Start with one hook: auto-format Python files on edit. It takes 30 seconds to set up and eliminates an entire class of code-review comments forever.
