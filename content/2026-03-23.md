## Quick Win — Apply This Today

**Set up your first `CLAUDE.md` for an active engagement.**

In any project directory you're currently working in, create a `CLAUDE.md` file at the root with:

```markdown
# Project: [Client Name / Engagement]

## Context
- What this project does in 2-3 sentences
- Key stakeholders and their expectations

## Tech Stack
- Languages, frameworks, data tools in use

## Conventions
- Naming conventions, file structure rules
- Any client-specific constraints (e.g., no external API calls, specific libraries only)

## Key Commands
- How to run tests, lint, start the data pipeline, etc.

## Off-Limits
- Files or systems Claude should not modify without explicit confirmation
```

Claude reads this automatically at the start of every session. Takes 5 minutes to write, saves you re-explaining context every time — and becomes useful documentation for the rest of the team.
