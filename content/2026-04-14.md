3. **Session management**: Agents maintain state across interactions. Start an analysis, pause it, resume hours later. The agent remembers where it was.

4. **Built-in security**: Process isolation, resource limits, network control, ephemeral filesystems. The enterprise governance story writes itself.

**What it costs:** $0.08 per runtime hour plus standard Claude model usage. An always-on agent is ~$58/month in runtime before tokens. For batch analysis that runs a few hours per week, the cost is negligible.

## Resource 2: Office Document Skills — The Analysis-to-Deliverables Pipeline

**Source:** [claude-office-skills — GitHub (tfriedel/claude-office-skills)](https://github.com/tfriedel/claude-office-skills)
**Also:** [10x Your Productivity with Claude PPTX Skill — SmartScope](https://smartscope.blog/en/generative-ai/claude/claude-pptx-skill-practical-guide/)
**Also:** [A Step-by-Step Guide to Claude's Excel, PowerPoint, and PDF Agent Skills](https://agentskills.so/blogs/technical-practice-document-automation)
**Time:** ~5 min (skim the GitHub README + SmartScope guide)
**Category:** Consulting Deliverables & Workflow Completion

### Why This Is Relevant

Every previous digest has focused on **analysis**: reading code, understanding architecture, generating documentation, verifying quality. But the consulting workflow doesn't end at markdown. It ends at a **PowerPoint deck on a partner's desk** or an **Excel workbook in a client's inbox**. This is the gap.

`claude-office-skills` is an open-source collection of Claude Code Skills for creating and editing PPTX, DOCX, XLSX, and PDF files. Install them, and Claude Code can:

- **Generate PowerPoint presentations** from analysis outputs — with proper slide layouts, charts, and formatting
- **Create Word documents** for detailed architecture reports
- **Build Excel workbooks** with structured data, pivot tables, and charts
- **Produce PDFs** for formal deliverables

**How it works:**

Each document format has its own `SKILL.md` that defines the workflow. When you ask Claude Code to create a presentation, it:

1. Checks for the relevant office skill
2. Reads the complete workflow from SKILL.md
3. Follows each step precisely (template selection, content placement, formatting)
4. Validates outputs
5. Organizes files into `outputs/<document-name>/`

**The compound effect with your existing workflow:**

Imagine this pipeline:

```
/reverse-engineer → architecture analysis (markdown + Mermaid)
    ↓
/create-pptx → client presentation (PowerPoint)
    ↓
/create-xlsx → data appendix (Excel with API inventory, risk register)
    ↓
Stop hook → verify all deliverables are complete and formatted
```

That's end-to-end: from raw codebase to client-ready deliverables in a single Claude Code session. No copy-pasting from terminal to PowerPoint. No manual formatting. The analysis and the deliverable are produced by the same agent in the same session.

**Included utilities:**

The repo includes Python scripts for post-processing: `thumbnail.py` (creates slide thumbnail grids for review), `rearrange.py` (reorders slides), `inventory.py` (extracts text inventory from presentations), and `replace.py` (batch text replacement from JSON). These are useful for template-based workflows where you generate a base deck and then customize per client.

**For ISP:** This completes the GenAI-in-SDLC story. The narrative becomes: *"Claude Code doesn't just analyze code — it produces the architecture documentation, the executive summary deck, and the risk register spreadsheet. The entire deliverable pipeline is automated."* For enterprise stakeholders who care about output, not process, this is the most compelling demo you can give.

### Key Takeaway

**Install `claude-office-skills` to close the gap between analysis and deliverables.** Your `/reverse-engineer` skill produces excellent markdown — but your clients want PowerPoint and Excel. These skills let Claude Code take the last step automatically. Clone the repo, copy the skills to `~/.claude/skills/`, and your next architecture analysis can end with a formatted slide deck instead of a terminal output.

## Quick Win: Install Office Skills and Connect Them to Your Pipeline (3 min)

**Step 1 — Clone the office skills repo (30 seconds):**

```bash
git clone https://github.com/tfriedel/claude-office-skills.git /tmp/office-skills
```

**Step 2 — Copy the PPTX and DOCX skills to your personal skills directory (30 seconds):**

```bash
mkdir -p ~/.claude/skills
cp -r /tmp/office-skills/public/pptx ~/.claude/skills/
cp -r /tmp/office-skills/public/docx ~/.claude/skills/
cp -r /tmp/office-skills/public/xlsx ~/.claude/skills/
```

**Step 3 — Test it (2 min):**

Open Claude Code and type:

```
Create a 5-slide executive summary presentation about a legacy Java codebase 
with 100K lines of code. Include: architecture overview, key services, 
API surface area, risk register, and recommended modernization roadmap.
```

Claude Code will detect the PPTX skill and follow the structured workflow to produce a formatted PowerPoint file. This is the same output your `/reverse-engineer` skill produces — but now as a client-ready slide deck instead of markdown.

**Why this is today's Quick Win:** You're heading into Office Hours with a 7-step workflow narrative. After the session, the first thing you'll want to do is prototype the end-to-end pipeline: `/reverse-engineer` → office skills → client deliverables. Having the skills pre-installed means you can start immediately. The 3-minute setup now saves you 30 minutes of context-switching later.
