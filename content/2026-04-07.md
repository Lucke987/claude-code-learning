## Resource 1: Hooks Deep Dive — The Official Guide

**Source:** [Automate Workflows with Hooks — Claude Code Docs](https://code.claude.com/docs/en/hooks-guide)
**Time:** ~7 min read
**Category:** Automation & Workflow Enforcement

### Why This Is Relevant
Previous digests touched on hooks briefly, but this is the comprehensive guide. Hooks are **deterministic** — unlike CLAUDE.md instructions that Claude *might* follow, hooks are shell commands that *always* execute at specific lifecycle points. For consulting work where compliance and consistency matter, this is critical.

**The 5 hook patterns most useful for consultants:**

1. **Desktop notifications** (`Notification` event): Get alerted when Claude finishes a long analysis instead of watching the terminal. Essential when running batch work with Auto Mode.

2. **Auto-format after edits** (`PostToolUse` with `Edit|Write` matcher): Run Prettier, Black, or any formatter automatically after every file edit. Client deliverables always match style guides.

3. **Block edits to protected files** (`PreToolUse`): Prevent Claude from modifying `.env`, config files, or client-sensitive data. Exit code 2 blocks the action and tells Claude why.

4. **Re-inject context after compaction** (`SessionStart` with `compact` matcher): Long sessions lose context when compacted. A hook that echoes "Reminder: this is Client X, use their API conventions" keeps Claude on track.

5. **Prompt-based hooks** (`type: "prompt"`): Use an LLM to make judgment calls — e.g., a `Stop` hook that checks "are all requested tasks actually complete?" before letting Claude finish.

**New hook types to know:**
- `type: "agent"` — spawns a sub-agent that can read files and run commands to verify conditions
- `type: "http"` — POSTs event data to a web endpoint (useful for team audit logging)
- `PermissionRequest` hooks — auto-approve specific tool calls you always allow

### Key Takeaway
**Start with the Notification hook** — it takes 30 seconds to set up and immediately improves your workflow with Auto Mode. Then add a `PostToolUse` formatter hook for your most common language. These two hooks alone eliminate the most common friction points in long Claude Code sessions.

## Quick Win: Set Up a Notification Hook Right Now (1 min)

Copy this into your `~/.claude/settings.json` (create the file if it doesn't exist):

```json
{
  "hooks": {
    "Notification": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "notify-send 'Claude Code' 'Claude Code needs your attention'"
          }
        ]
      }
    ]
  }
}
```

*(Replace `notify-send` with `osascript -e 'display notification \"Claude Code needs your attention\" with title \"Claude Code\"'` on macOS.)*

Now every time Claude finishes a task or needs your input, you get a desktop notification. This is especially powerful combined with Auto Mode — kick off a batch analysis, switch to another task, and get pinged when Claude is done.

**Next step:** Once that's working, add a `PostToolUse` hook to auto-format files. Check Resource 1 for the Prettier example, or ask Claude: *"Add a PostToolUse hook that runs black on every Python file I edit."*

The bigger picture: Hooks are the "set it and forget it" enforcement layer. CLAUDE.md tells Claude what to do; hooks **make sure** it happens. Combined with Auto Mode and `/loop` from yesterday, you now have the full autonomous workflow stack: Claude runs tasks on schedule, executes without interruption, formats and validates automatically, and notifies you when done.
