### 2. Claude Code Common Workflows — Official Docs

**Source:** [code.claude.com/docs/en/common-workflows](https://code.claude.com/docs/en/common-workflows)
**Time:** ~5 min (skim the sections most relevant to you)

**Why it's relevant:** This is the definitive reference for Claude Code workflow patterns. Key sections for your consulting work:
- **Parallel sessions with Git worktrees** (`claude --worktree feature-auth`) — run multiple tasks simultaneously without conflicts
- **Unix-style piping** (`cat data.csv | claude -p 'analyze this'`) — integrate Claude into shell scripts for batch data processing
- **Plan Mode** (Shift+Tab) — have Claude analyze a codebase or dataset read-only before making changes, perfect for understanding a new client's code or data
- **Hooks for notifications** — get desktop alerts when Claude finishes a long-running analysis so you can context-switch to other work

**Key Takeaway:** Use `claude --worktree` to run parallel sessions: one analyzing data while another builds the deliverable. This alone can cut your turnaround time significantly.

## Quick Win: Apply This Today

**Set up a "deck-from-data" workflow in under 2 minutes:**

1. Install the [claude-office-skills](https://github.com/tfriedel/claude-office-skills) package
2. Place a CSV or Excel file with your analysis data in your working directory
3. Run: `claude -p "Read data.csv, identify the top 3 insights, and create a 5-slide executive summary deck as PPTX"`
4. Review and refine the output: `claude --continue` → "Make the charts on slide 3 use the BCG color palette"

This turns a 30-minute manual process into a 2-minute conversational workflow — and you can save the prompt as a custom command for reuse across projects.
