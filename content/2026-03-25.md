### 1. Unix-style piping and CLI automation with Claude Code
**URL:** https://code.claude.com/docs/en/common-workflows#use-claude-as-a-unix-style-utility
**Time:** ~6 min read

**Why it's relevant to you:**
As a BCG consultant, you're constantly processing outputs — error logs, build results, CSVs, API responses, model outputs. Claude Code's `-p` flag makes it a composable Unix utility you can drop into any shell pipeline, turning ad-hoc analysis into a one-liner.

**Key Takeaway:**
Pipe data directly into Claude and get structured output back:
```bash
# Summarise a dataset anomaly in seconds
tail -200 app.log | claude -p "flag any anomalies and explain root causes"

# Turn unstructured text into structured JSON for downstream processing
cat client-notes.txt | claude -p "extract action items as JSON array" --output-format json > actions.json

# Use as a smart linter in your scripts
git diff main --name-only | claude -p "review changed files for security issues"
```
The `--output-format json` flag is particularly powerful when chaining Claude into data pipelines — you get parseable output, not prose.

### 2. Claude Code Hooks — zero-exception automation
**URL:** https://code.claude.com/docs/en/hooks
**Time:** ~7 min read

**Why it's relevant to you:**
CLAUDE.md instructions are advisory — Claude *tries* to follow them. Hooks are deterministic. If you want something to *always* happen (auto-format after every file edit, block writes to sensitive folders, send a desktop notification when a long task finishes), hooks are the right tool.

**Key Takeaway:**
Four hook types (`command`, `http`, `prompt`, `agent`) fire at lifecycle points (`PreToolUse`, `PostToolUse`, `Stop`, `Notification`, etc.). The quickest wins:

```json
// ~/.claude/settings.json — desktop notification when Claude finishes
{
  "hooks": {
    "Notification": [{
      "matcher": "idle_prompt",
      "hooks": [{
        "type": "command",
        "command": "osascript -e 'display notification \"Claude is done\" with title \"Claude Code\"'"
      }]
    }],
    "PostToolUse": [{
      "matcher": "Edit",
      "hooks": [{
        "type": "command",
        "command": "cd $CLAUDE_PROJECT_DIR && npm run lint --silent 2>&1 | head -20"
      }]
    }]
  }
}
```
Ask Claude to write hooks for you: *"Write a hook that runs black after every Python file edit"* — it will generate the JSON directly.

### 3. CLAUDE.md + Auto Memory — persistent context that compounds
**URL:** https://code.claude.com/docs/en/memory
**Time:** ~8 min read

**Why it's relevant to you:**
Every Claude Code session starts with a blank context window. Without CLAUDE.md, you're re-explaining your project, data formats, and preferences every single session. With it, Claude already knows your conventions on day 100 the same as day 1. Auto memory extends this further — Claude writes its own learnings (build commands, debugging insights, discovered patterns) without you lifting a finger.

**Key Takeaway:**
Run `/init` in any project to auto-generate a starter CLAUDE.md. Then keep it lean:

```markdown
# CLAUDE.md — keep under 200 lines

## Build & test
- Run `python -m pytest tests/ -x` to run tests (stop on first failure)
- Lint: `ruff check . && mypy src/`

## Code style
- Type hints required on all functions
- Prefer `pathlib.Path` over `os.path`
- Data files live in `data/raw/` (read-only), outputs go to `data/processed/`

## Workflow
- Never commit to main directly — branch from `dev`
- Before any analysis script, validate input schema with `schema.validate(df)`
```

**Auto memory bonus:** when Claude notices you always use `pnpm` not `npm`, or that your test suite needs a local Redis instance, it saves that to `~/.claude/projects/<repo>/memory/MEMORY.md` automatically. Run `/memory` to audit and edit what it has learned.

### 4. Plan Mode + Subagents — explore without burning context
**URL:** https://code.claude.com/docs/en/common-workflows#use-plan-mode-for-safe-code-analysis
**Time:** ~4 min read

**Why it's relevant to you:**
The single biggest Claude Code productivity leak for heavy users is context pollution — long sessions where failed attempts, read files, and tangents crowd out working memory. Plan Mode + subagents solve this structurally.

**Key Takeaway:**
Two habits to adopt immediately:

1. **Start complex tasks in Plan Mode** (`Shift+Tab` twice, or `claude --permission-mode plan`). Claude reads files and reasons without making changes. Press `Ctrl+G` to open the plan in your editor and refine it before execution. This prevents Claude from solving the wrong problem at file 50.

2. **Delegate research to subagents** to protect your main context:
   ```
   Use subagents to investigate how our data ingestion pipeline handles schema mismatches,
   and whether there's an existing validation utility I should reuse.
   ```
   The subagent reads dozens of files, summarises findings, and reports back — without any of that file content entering *your* session's context window.

### 5. Scheduled tasks — automate recurring work
**URL:** https://code.claude.com/docs/en/web-scheduled-tasks
**Time:** ~4 min read (skim)

**Why it's relevant to you:**
If you find yourself running the same Claude task at the start of each day (e.g., PR reviews, dependency audits, log summaries, weekly report drafts), you can schedule it once and get results in your inbox — even when your laptop is off.

**Key Takeaway:**
Three scheduling options based on where the task runs:

| Option | Where it runs | Best for |
|--------|--------------|----------|
| Cloud scheduled tasks (`/schedule`) | Anthropic infrastructure | Tasks that run overnight/off-hours |
| Desktop scheduled tasks | Your machine | Tasks needing local file access |
| `/loop` in CLI | Current session | Quick polling while you're working |

Start a cloud scheduled task from the web at [claude.ai/code](https://claude.ai/code) or by running `/schedule` in any CLI session. Write the prompt as if Claude can't ask clarifying questions — be explicit about success criteria and what to do with results.

## Quick Win

**Apply this today in under 5 minutes:**

Set up a desktop notification hook so you know when Claude finishes a long task. Add this to `~/.claude/settings.json`:

```json
{
  "hooks": {
    "Notification": [{
      "matcher": "",
      "hooks": [{
        "type": "command",
        "command": "osascript -e 'display notification \"Claude Code needs your attention\" with title \"Claude Code\"'"
      }]
    }]
  }
}
```

*(Linux: replace `osascript` command with `notify-send 'Claude Code' 'Needs attention'`)*

Then kick off a long Claude task, switch to another window, and let Claude come to you. No more tab-switching to check if it's done.
