## Resource 1: /loop — Put Claude Code on Autopilot for Recurring Tasks

**Source:** [Put Claude on Autopilot: Scheduled Tasks with /loop — Medium](https://medium.com/@richardhightower/put-claude-on-autopilot-scheduled-tasks-with-loop-and-schedule-built-in-skills-43f3be5ac1ec)
**Time:** ~5 min read
**Category:** Automation & Scheduling

### Why This Is Relevant
As a consultant, you have tasks that repeat: checking if a data pipeline ran successfully, monitoring a deployment, refreshing a client dashboard, or polling an API for updated metrics. The `/loop` command lets you schedule these inside a Claude Code session with one line — no external cron jobs, no separate scripts.

How it works: `/loop 30m check if the ETL pipeline completed and summarize any errors` runs every 30 minutes while your session is open. You can use intervals in seconds (`s`), minutes (`m`), hours (`h`), or days (`d`). Claude runs the task in the background and notifies you when something needs attention.

**Constraints to know:** Tasks only run while Claude Code is active and idle. They auto-expire after 7 days. For durable scheduling that survives restarts, use GitHub Actions or Desktop scheduled tasks instead.

### Key Takeaway
**Use `/loop` for session-scoped monitoring tasks** — e.g., `/loop 1h check the latest data export in ./output/ and flag any rows with missing values`. This turns Claude into a background analyst that watches your data while you focus on client-facing work. For anything that needs to survive a terminal close, pair it with a GitHub Action.

## Resource 2: Auto Mode — Safe Unattended Execution for Batch Work

**Source:** [Claude Code Auto Mode: Safe Uninterrupted Development — ClaudeFast](https://claudefa.st/blog/guide/development/auto-mode) | [Auto Mode Security Analysis — Michael Bargury](https://www.mbgsec.com/archive/2026-03-29-claude-code-auto-mode-a-safer-way-to-skip-permissions/)
**Time:** ~5 min read (skim both)
**Category:** Productivity & Security

### Why This Is Relevant
Every time Claude pauses for a permission prompt ("Allow this bash command?"), it breaks your flow. Auto Mode, launched March 24, 2026, uses a background AI classifier to evaluate each tool call before execution — safe operations proceed automatically, while dangerous ones (deleting files, running unknown scripts) are still blocked. This is ideal for batch work like generating multiple deliverables, running analysis across several datasets, or processing a backlog of client files.

For headless/CI scenarios (e.g., a nightly data refresh running via `claude -p`), Auto Mode is essential since there's no human to approve prompts. Pair it with a sandboxed environment for safety.

### Key Takeaway
**Enable Auto Mode for your next batch analysis session**: press `Shift+Tab` twice (or type `auto` in the permission prompt) to activate it. Claude will breeze through read/write/execute steps without pausing. For client-sensitive work, run it in a sandboxed container or a dedicated Git worktree to limit the blast radius. Auto Mode + worktrees = parallel unattended analysis pipelines.

## Resource 3: CLAUDE.md Best Practices — Your Project's AI Onboarding Doc

**Source:** [CLAUDE.md Best Practices: 10 Sections to Include — UX Planet](https://uxplanet.org/claude-md-best-practices-1ef4f861ce7c?gi=77cc11fce3f9)
**Time:** ~4 min read
**Category:** Project Setup & Team Workflow

### Why This Is Relevant
You've now set up hooks, custom commands, MCP servers, and sub-agents across your projects. But CLAUDE.md is what ties them all together — it's the single file that tells Claude Code how your project works, what commands to run, and what conventions to follow. For a consultant managing multiple client projects, a well-structured CLAUDE.md per project means Claude instantly "knows" each client's stack, coding standards, and workflow when you switch contexts.

Key insight: **keep it under 200 lines** and use "progressive disclosure" — don't dump everything Claude might need; instead, tell it *where to find* information (e.g., "see docs/ for API specs"). Link to your custom commands and sub-agent definitions so Claude discovers them automatically.

### Key Takeaway
**Create a CLAUDE.md template you reuse across client projects** with these sections: (1) Project overview & stack, (2) Key directories & file conventions, (3) Build/test/lint commands, (4) Code style rules, (5) Links to custom commands and sub-agents. Check it into Git so your team benefits too. A 100-line CLAUDE.md eliminates the first 10 minutes of "getting Claude up to speed" every time you open a project.

## Quick Win: Set Up a /loop Monitor for Your Current Project (1 min)

If you have any recurring check in your current work — a build status, a data file that gets updated, a log to watch — try this right now:

```
/loop 15m check git log --oneline -5 and summarize what changed since last check
```

Or for data work:

```
/loop 30m ls -la ./output/ and tell me if any new files appeared or if file sizes changed
```

This gives you a "background assistant" that watches for changes while you focus on higher-value work. You can list active loops with `/schedule list` and cancel any with `/schedule cancel <id>`.

The bigger picture: combine today's `/loop` with the MCP servers from yesterday's digest and the sub-agents from the day before, and you have a fully autonomous consulting workflow — Claude monitors data sources (MCP), runs analyses on schedule (/loop), delegates to specialized agents (sub-agents), and does it all without permission interruptions (Auto Mode).
