### 1. New This Week: The `/recap` Feature & April Changelog Highlights

**Source:** [Claude Code Changelog](https://code.claude.com/docs/en/changelog)
**Time:** ~3 min read
**Why it's relevant:** Dropped yesterday (April 14), the `/recap` feature automatically summarizes your session context when you return — perfect for picking up client work across days. The `/resume` picker now also defaults to sessions in your current directory (Ctrl+A for all projects).

**Other April highlights:**

- Write tool is 60% faster on files with tabs/special characters
- Subprocess sandboxing with PID namespace isolation on Linux

**Key Takeaway:** Start using `/recap` today. When you come back to a client project after a break, type `/recap` instead of re-explaining your context. It saves 2-3 minutes per session restart.

### 2. Claude Code Power Tips for Data Work

**Source:** [Claude Code Power Tips — KDnuggets](https://www.kdnuggets.com/claude-code-power-tips)
**Time:** ~5 min read
**Why it's relevant:** Written specifically for data professionals. Covers the exact workflow a consultant uses daily: profiling CSVs, cleaning data, building pipelines, and iteratively refining analysis scripts.

**Top tips from the article:**
- Use `@` to attach data files directly (e.g., `@customer_data.csv`) so Claude reads the full content
- Activate **Plan Mode** before multi-file refactors — Claude proposes a step-by-step plan you review before execution
- Enable **Extended Thinking** for complex problems (slow transforms, model debugging)
- Use iterative prompting: get initial code, then ask to "optimize for speed" or "add validation"

**Key Takeaway:** Next time you profile a client data set, try: `"Analyze @data.csv. Provide summary statistics, check for missing values, and identify potential outliers in numerical columns."` — one prompt replaces 15 minutes of manual EDA.

## Quick Win: Apply This Immediately

**Set up `/recap` for your current project:**

```bash
# 1. Update Claude Code to latest (v2.1.109+)
claude update

# 2. Open your current client project
cd ~/your-project

# 3. When returning to a session, just type:
/recap

# 4. Or resume a previous session (now defaults to current directory):
/resume
```

This alone saves 2-3 minutes every time you context-switch between client projects — which, as a consultant juggling multiple engagements, likely happens several times a day. Over a week, that's 15-30 minutes reclaimed just from this one feature.
