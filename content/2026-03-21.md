### 2. Writing a Good CLAUDE.md
**URL:** https://www.humanlayer.dev/blog/writing-a-good-claude-md
**Time:** ~5 min read
**Why it's relevant:** CLAUDE.md is the single highest-leverage file in your repo. Getting it right means every Claude Code session starts with full project context — no re-explaining your stack, conventions, or data structure.
**Key Takeaway:** Less is more. Every instruction you add dilutes the ones above it — LLMs start ignoring *all* instructions as count increases. Keep CLAUDE.md to universally applicable rules only (key commands, architecture map, coding conventions). Use hooks for anything that must happen deterministically (linting, formatting, tests).

## Quick Win

**Apply this today in under 5 minutes:**

Create a `CLAUDE.md` file at the root of your current project with just 3 things:

```markdown
# Project Context
[One sentence: what this project does and for which client/domain]

## Key Commands
- Run analysis: `python src/main.py`
- Run tests: `pytest tests/`

## Data Structure
- Raw data: `data/raw/`
- Outputs: `data/outputs/`, figures in `figures/`
- Key tables: [list your main dataframes/tables and what they represent]
```

That's it. Next time you open Claude Code in this project, it will have full context from the first message — no warm-up needed. You'll notice it makes better decisions about file paths, avoids redundant data loading, and produces cleaner first-pass outputs immediately.
