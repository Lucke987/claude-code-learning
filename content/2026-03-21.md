## Quick Win — Apply This Today

**Set up your first hook in under 5 minutes.**

Create or open `.claude/settings.json` in your project root and add:

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "echo \"$(date): edited $CLAUDE_TOOL_INPUT_PATH\" >> ~/.claude_activity.log"
          }
        ]
      }
    ]
  }
}
```

This creates a lightweight audit log every time Claude edits a file — a useful starting point for understanding your own usage patterns, and a foundation for more sophisticated hooks (auto-formatting, git snapshots, Slack notifications). Run `/hooks` inside Claude Code to verify it's active.

> **Why this matters for BCG work:** On client engagements, knowing exactly which files Claude touched (and when) gives you a clean audit trail — useful for handoffs, QA, and understanding what changed in a complex session.
