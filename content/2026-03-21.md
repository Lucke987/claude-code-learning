CLAUDE.md is the single highest-leverage file in your Claude Code setup. For a consultant running Python analysis scripts, data pipelines, or automation tools, a well-crafted CLAUDE.md means Claude always knows your project structure, naming conventions, and exact test commands — without you repeating yourself every session.

**Key Takeaway:**
Include five things that matter most: (1) one-liner project description, (2) exact commands to run tests/build/lint, (3) code style preferences with *specific* examples, (4) gotchas and known pitfalls, and (5) any external service dependencies. Ruthlessly prune the rest — every line competes with your actual work for Claude's attention. Never put code style enforcement here; use a linter instead.
