### 2. Claude Code Power User Tips (Official Anthropic Help Center)

**Link:** [https://support.claude.com/en/articles/14554000-claude-code-power-user-tips](https://support.claude.com/en/articles/14554000-claude-code-power-user-tips)
**Time:** ~5 min read
**Category:** Productivity — Daily Workflow

**Why it's relevant:** This is the official power-user reference covering three high-leverage features you should be using in every session: (1) **CLAUDE.md** as your project's "constitution" so Claude understands your repo without re-explanation, (2) **Slash commands** in `.claude/commands/` for anything you do more than once a day, and (3) **Hooks** for deterministic actions that must happen every time (linting, formatting, security checks). The single most impactful tip: give Claude a way to verify its own output.

**Key Takeaway:** Create a CLAUDE.md file for your most active project today. Include your folder structure, coding conventions, and common commands. Every future session will be faster and more accurate because of it.

## Quick Win

**Set up a `/analyze` slash command for instant data profiling.**

Create the file `.claude/commands/analyze.md` in your most-used project with this content:

```
Load the file at $ARGUMENTS. Profile it: shape, column types, missing values,
basic statistics, and outlier detection. Then suggest 3 visualizations for the
most interesting patterns. Output a summary table first, then the suggestions.
```

Now every time you get a new dataset, just type `/analyze path/to/data.csv` and get a full profile in seconds. This is the kind of small automation that saves 10-15 minutes per analysis — and compounds across dozens of datasets per engagement.
