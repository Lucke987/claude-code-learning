### 3. CLAUDE.md Best Practices - Make Every Session Start Smart

**Source:** [50 Claude Code Tips and Best Practices For Daily Use](https://www.builder.io/blog/claude-code-tips-best-practices)

**Time:** ~4 min read (focus on the CLAUDE.md section)

**Why it's relevant:** As a daily Claude Code user, the single highest-ROI investment is a well-crafted CLAUDE.md file. This resource covers how to structure project memory so Claude Code starts every session with the right context - your data schemas, naming conventions, preferred chart styles, and analysis patterns.

**Key Takeaway:** Keep your CLAUDE.md under 200 lines and focused. For consulting projects, include: (1) data source descriptions, (2) client-specific terminology, (3) preferred output formats, and (4) common analysis patterns. Use `~/.claude/CLAUDE.md` for global consulting defaults and per-project `CLAUDE.md` for client-specific context.

## Quick Win - Apply This Today

**Set up a global `~/.claude/CLAUDE.md` for your consulting work.** Create the file with 10-15 lines covering:

```markdown
# Global Consulting Defaults
- Default to Python with pandas for data analysis
- Use seaborn/matplotlib for visualizations with clean, presentation-ready styling
- When creating charts, always include titles, axis labels, and source annotations
- Output data summaries as formatted markdown tables
- For Excel exports, include a summary sheet with key metrics
- Prefer concise, insight-driven commentary over technical descriptions
```

This takes 2 minutes to set up and will improve every future Claude Code session. Then run `/powerup` to discover what else you've been missing.
